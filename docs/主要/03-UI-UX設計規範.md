# BBScoring — UI/UX 設計規範

## 1. 設計原則

### 1.1 核心理念
- **速度優先**：記錄員需要在數秒內完成一個事件的記錄
- **觸控友善**：所有操作針對觸控螢幕最佳化
- **一目了然**：比賽狀態在任何時刻都能快速掌握
- **容錯寬鬆**：方便修改，降低記錄壓力

### 1.2 設計目標
| 目標 | 指標 |
|------|------|
| 記錄一球結果 | ≤ 1 次點擊 |
| 記錄完整打席（三振/保送） | 自動完成（投球累計） |
| 記錄打擊結果 + 方向 | ≤ 3 次點擊 |
| 修改歷史記錄 | ≤ 2 次點擊抵達 |
| 切換顯示面板 | 滑動 or 1 次點擊 |

### 2.3 色彩使用規則
- 好球用**紅色**，壞球用**綠色**（符合國際慣例）
- 得分相關用**綠色系**，出局相關用**紅色系**
- 安打用**藍色系**，全壘打用**紫色**特殊標示
- 失誤用**橘色**明顯區分

## 6. 互動流程

### 6.1 一般投球流程
```
記錄員                        系統
  │                            │
  │  點擊 [好球]               │
  │ ──────────────────────────▶│
  │                            │ 更新球數 S:0→1
  │                            │ 振動回饋 (50ms)
  │  ◀──────────────────────── │ 顯示更新後球數
  │                            │
  │  點擊 [壞球]               │
  │ ──────────────────────────▶│
  │                            │ 更新球數 B:0→1
  │  ◀──────────────────────── │ 顯示更新後球數
  │                            │
  │  點擊 [好球]               │
  │ ──────────────────────────▶│
  │                            │ 更新球數 S:1→2
  │  ◀──────────────────────── │ 顯示更新後球數
  │                            │
  │  點擊 [打出去]             │
  │ ──────────────────────────▶│
  │                            │ 滑出打擊結果面板
  │  ◀──────────────────────── │
  │                            │
  │  點擊 [一壘安打]           │
  │ ──────────────────────────▶│
  │                            │ 標記打擊結果
  │                            │
  │  點擊球場圖 [左外野方向]    │
  │ ──────────────────────────▶│
  │                            │ 標記打擊方向
  │                            │
  │  點擊 [完成]               │
  │ ──────────────────────────▶│
  │                            │ 收合面板
  │                            │ 更新壘上跑者
  │                            │ 自動儲存
  │  ◀──────────────────────── │ 切換下一打者
```

### 6.2 三振流程（自動判定）
```
  S:0 → [好球] → S:1
  S:1 → [壞球] → B:1
  B:1 → [界外] → S:2 (界外不計第三好球)
  S:2 → [揮空] → S:3 → 系統自動判定三振 → 出局+1 → 下一打者
```

### 6.3 保送流程（自動判定）
```
  B:0 → [壞球] → B:1
  B:1 → [壞球] → B:2
  B:2 → [好球] → S:1
  S:1 → [壞球] → B:3
  B:3 → [壞球] → B:4 → 系統自動判定保送 → 打者上一壘 → 下一打者
```

### 6.4 修改歷史記錄流程
```
  1. 滑動至歷史面板（或點擊歷史標籤）
  2. 看到逐球記錄列表
  3. 點擊要修改的記錄
  4. 彈出編輯面板（與新增記錄相同介面）
  5. 修改後點擊 [儲存修改]
  6. 系統重新計算後續影響
  7. 修改過的記錄顯示黃色標記
```

## 7. 動畫與過渡

### 7.1 過渡效果
| 動作 | 效果 | 時長 |
|------|------|------|
| 面板滑入/滑出 | slide + fade | 300ms |
| 按鈕按壓 | scale(0.95) | 150ms |
| 球數更新 | 圓點填色動畫 | 200ms |
| 得分 | 數字彈跳 + 閃爍 | 500ms |
| 壘上跑者移動 | 位移動畫 | 400ms |
| 出局 | 漸隱 + 下滑 | 300ms |
| 撤銷操作 | 反向動畫 | 250ms |

### 7.2 動畫原則
- 使用 `transform` 和 `opacity` 做動畫（GPU 加速）
- `will-change` 僅在動畫前設置
- 省電模式下停用所有動畫
- `prefers-reduced-motion` 媒體查詢停用動畫

## 8. 觸控規範

### 8.1 觸控目標最小尺寸
| 元素類型 | 最小尺寸 | 建議尺寸 |
|----------|----------|----------|
| 主要按鈕 | 48×48px | 60×60px |
| 次要按鈕 | 44×44px | 48×48px |
| 球場點擊區域 | 36×36px | 44×44px |
| 壘包 | 44×44px | 56×56px |
| 列表項目 | 高44px | 高56px |

### 8.2 間距規範
- 按鈕之間最小間距：8px
- 觸控區域不可重疊
- 邊緣留白至少 16px

## 9. 無障礙設計

- 所有按鈕含 `aria-label` 說明
- 球數使用顏色 + 形狀雙重線索（不只以顏色區分）
- 高對比度配色（WCAG AA 標準）
- 螢幕閱讀器友好的語意化標記
- 動畫可關閉
