# BBScoring — 功能規格細節

## 1. 首頁（比賽列表）

### 1.1 頁面說明
應用程式開啟時的首頁，展示所有比賽記錄，並提供新增比賽入口。

### 1.2 頁面元素
```
┌──────────────────────────────────────┐
│  ⚾ BBScoring                   ⚙   │
├──────────────────────────────────────┤
│                                      │
│  ┌──────────────────────────────┐   │
│  │ 🔴 進行中                     │   │
│  │ 112年春季聯賽 第3場           │   │
│  │ 猛虎 3 - 2 飛鷹  │ 5局上     │   │
│  │ 2026-03-17 14:00             │   │
│  │                    [繼續記錄]  │   │
│  └──────────────────────────────┘   │
│                                      │
│  ┌──────────────────────────────┐   │
│  │ ✅ 已完成                     │   │
│  │ 112年春季聯賽 第2場           │   │
│  │ 猛虎 5 - 3 雄鷹              │   │
│  │ 2026-03-10 14:00             │   │
│  │              [查看] [匯出]    │   │
│  └──────────────────────────────┘   │
│                                      │
│         ┌──────────────┐            │
│         │   ＋ 新增比賽  │            │
│         └──────────────┘            │
│                                      │
└──────────────────────────────────────┘
```

### 1.3 功能規格
| 功能 | 操作 | 說明 |
|------|------|------|
| 新增比賽 | 點擊「＋ 新增比賽」 | 導向比賽設定頁 |
| 繼續記錄 | 點擊進行中比賽的「繼續記錄」 | 導向即時記錄頁 |
| 查看比賽 | 點擊已完成比賽的「查看」 | 導向統計/回顧頁 |
| 匯出比賽 | 點擊「匯出」 | 下載 JSON 檔案 |
| 刪除比賽 | 長按比賽卡片 | 確認後刪除 |
| 匯入比賽 | 設定頁面中操作 | 選擇 JSON 檔案匯入 |

## 2. 比賽設定頁

### 2.1 設定流程（引導式精靈）
```
Step 1: 比賽基本資訊
         ↓
Step 2: 客隊設定（隊名 + 球員名單）
         ↓
Step 3: 主隊設定（隊名 + 球員名單）
         ↓
Step 4: 客隊打序 & 先發守備
         ↓
Step 5: 主隊打序 & 先發守備
         ↓
Step 6: 確認 → 開始比賽
```

## 3. 即時記錄頁（核心功能）

### 3.1 頂部狀態列
永遠可見，顯示最關鍵資訊：
```
┌────────────────────────────────────────────────┐
│ 猛虎 3 │ 5局上 │ 1出局 │ S:2 B:1 │ ⟲  ⟳  ≡  │
│ 飛鷹 2 │       │       │         │            │
└────────────────────────────────────────────────┘
```

- 比分：雙方隊名 + 分數
- 局數：第幾局 + 上/下半
- 出局數：目前出局數
- 球數：S (好球) B (壞球) 簡顯
- 工具：撤銷、重做、選單

**操作順序建議（非強制）：**
1. 選擇打擊結果（必填）
2. 點擊球場圖選擇方向（選填）
3. 選擇球質（選填）
4. 設定守備路徑（選填）
5. 處理失誤（若有）
6. 確認跑壘異動（系統自動推算，可手動調整）
7. 點擊「完成」

### 3.4 球數自動判定

| 狀態 | 觸發條件 | 系統動作 |
|------|----------|----------|
| 三振出局 | 3 好球（非界外） | 自動記錄三振，出局 +1 |
| 四壞保送 | 4 壞球 | 自動記錄保送，打者上一壘 |
| 三出局 | 3 出局 | 自動切換半局 |
| 第三好球界外觸擊 | 2好球 + 界外觸擊 | 自動記錄三振（界外觸擊三振） |

### 3.5 跑壘自動推算

系統會根據打擊結果自動推算跑壘：

| 打擊結果 | 自動推算 |
|----------|----------|
| 一壘安打 | 打者上一壘；原壘上跑者各進一壘（若強迫） |
| 二壘安打 | 打者上二壘；原壘上跑者各進兩壘（可手動調整） |
| 三壘安打 | 打者上三壘；原壘上跑者全部得分 |
| 全壘打 | 打者 + 所有跑者全部得分 |
| 滾地出局 | 打者出局；跑者依守備路徑判斷 |
| 四壞保送 | 打者上一壘；強迫進壘的跑者進一壘 |

**手動調整：**
- 自動推算後，記錄員可點擊壘包圖上的跑者修改位置
- 例如：一壘安打但跑者從一壘跑到三壘，可手動調整

### 3.6 特殊事件處理

#### 暴投（WP）/ 捕逸（PB）
```
1. 記錄員點擊 [暴投] 或 [捕逸]
2. 系統顯示壘包圖，標示可推進的跑者
3. 記錄員點擊要推進的跑者
4. 選擇推進到哪個壘包
5. 球數根據暴投/捕逸規則更新（壞球 +1 或不變）
```

#### 盜壘（投球前或投球間）
```
1. 記錄員長按壘包上的跑者
2. 彈出選單：[盜壘成功] [盜壘失敗] [牽制出局]
3. 選擇結果
4. 系統更新壘包狀態
```

#### 投手更換
```
1. 點擊頂部狀態列的「≡」選單
2. 選擇「更換投手」
3. 從替補名單選擇新投手
4. 確認更換
5. 系統記錄更換時間點，壘上跑者自責分歸屬
```

#### 代打/代跑/代守
```
1. 點擊「≡」選單 → 「替換球員」
2. 選擇替換類型：代打 / 代跑 / 代守
3. 選擇被替換的球員
4. 選擇替補球員
5. 確認守備位置（代守時）
6. 確認替換
```

### 6.2 修改功能
- 點擊任一打席記錄 → 開啟編輯模式
- 編輯模式與新記錄介面一致
- 修改後的記錄以黃色邊框標示
- 修改會觸發後續狀態重新計算

### 6.3 篩選功能
- 按局數篩選
- 按隊伍篩選
- 按事件類型篩選（安打、出局、失誤…）
- 按球員篩選

## 10. 錯誤處理與提示

### 10.1 操作確認
| 操作 | 確認方式 |
|------|----------|
| 刪除比賽 | 彈出確認對話框：「確定刪除此比賽記錄？此操作不可復原。」 |
| 結束比賽 | 彈出確認：「確定結束比賽？結束後仍可查看和匯出記錄。」 |
| 清除所有資料 | 二次確認：先確認 → 再輸入「確認刪除」文字 |
| 替換球員 | 顯示替換摘要確認 |

### 10.2 異常提示
| 情境 | 提示 |
|------|------|
| 打序未設定完整 | 「請先完成 9 位先發球員的設定」 |
| 無投手 | 「請先指定先發投手」 |
| 重複背號 | 「背號 #7 已有球員使用」 |
| LocalStorage 空間不足 | 「儲存空間不足，請匯出並清理舊比賽」 |
| 匯入檔案格式錯誤 | 「檔案格式不正確，請選擇 BBScoring 匯出的 JSON 檔案」 |
