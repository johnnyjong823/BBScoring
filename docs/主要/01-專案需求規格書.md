# BBScoring — 棒球比賽即時記錄系統 需求規格書

## 1. 專案概述

### 1.1 專案名稱
BBScoring（Baseball Scoring）— 棒球比賽即時記錄系統

### 1.2 專案目標
建立一個**純前端**（HTML + CSS + JavaScript）的棒球比賽記錄網站，讓記錄員能在**平板或手機**上快速、精確地記錄整場棒球比賽的每一個細節，包含打擊方向、守備失誤、暴投、觸身球等所有事件。

### 1.3 目標使用者
- 棒球比賽記錄員（主要）
- 教練團隊（查看記錄）
- 球隊管理者（查看統計）

### 1.4 使用情境
- 記錄員在球場旁邊，手持**平板**或**手機**進行即時記錄
- 需要在戶外強光環境下清晰可見
- 操作必須快速，一個打席的記錄在數秒內完成
- 記錄可來回修改、補正

### 2.1 技術堆疊
| 項目 | 技術 |
|------|------|
| 前端結構 | HTML5 |
| 樣式設計 | CSS3（含 RWD） |
| 邏輯處理 | Vanilla JavaScript（ES6+） |

### 2.2 支援裝置與瀏覽器
| 項目 | 支援範圍 |
|------|----------|

| 瀏覽器 | Chrome、Safari、Edge（最近兩個版本） |
| 螢幕方向 | 橫向（平板優先）、直向（手機適配） |

### 2.3 RWD 斷點設計（Mobile First）
```
手機直向：     < 576px
手機橫向：     576px ~ 767px
平板直向：     768px ~ 1023px
平板橫向：     1024px ~ 1365px
桌面（備援）： ≥ 1366px
```

### 2.4 效能目標
- 首次載入 < 3 秒（含快取）
- 操作回應 < 100ms
- 離線可用（PWA）
- 資料不丟失（自動儲存）

## 4. 比賽設定模組

### 4.1 建立新比賽
- 比賽日期、時間
- 比賽名稱（例：「112年甲組春季聯賽 第3場」）
- 主場 / 客場隊伍名稱
- 比賽局數設定（預設 9 局，可設 5、7、9 局）
- 比賽場地
- 備註欄位

### 4.2 隊伍與球員管理
- 每隊可登錄 25 人球員名單
- 球員資訊：
  - 背號（必填）
  - 姓名（必填）
  - 投打習慣（左投/右投、左打/右打/左右開弓）
  - 守備位置（可複選）
  - 備註

### 4.3 打序設定
- 先發 9 人打序排列（拖曳或按鈕調整順序）
- 指定先發守備位置
- 指定先發投手
- 替補名單

## 5. 逐球記錄模組（核心）

### 5.1 每球記錄流程
```
1. 顯示目前局數、半局、打者、投手、球數
2. 記錄員按下該球結果
3. 系統自動更新球數（S-B-O）
4. 若產生打擊結果 → 進入打擊結果記錄
5. 系統自動推進比賽狀態
```

### 5.2 每球結果分類

#### A. 未打擊出去
| 結果 | 代碼 | 說明 |
|------|------|------|
| 好球（稱好球） | S | Called Strike |
| 揮空 | SS | Swinging Strike |
| 壞球 | B | Ball |
| 界外球 | F | Foul Ball（兩好球後不計） |
| 界外觸擊 | FB | Foul Bunt |
| 觸身球 | HBP | Hit By Pitch |
| 故意四壞 | IBB | Intentional Base on Balls |
| 暴投 | WP | Wild Pitch |
| 捕逸 | PB | Passed Ball |
| 投手犯規 | BK | Balk |

#### B. 打擊出去（進入打擊結果子流程）
| 結果 | 代碼 | 說明 |
|------|------|------|
| 比賽進行中 | IP | In Play |

### 5.3 打擊結果記錄

#### 安打類型
| 類型 | 代碼 | 說明 |
|------|------|------|
| 一壘安打 | 1B | Single |
| 二壘安打 | 2B | Double |
| 三壘安打 | 3B | Triple |
| 全壘打 | HR | Home Run |
| 內野安打 | IH | Infield Hit |
| 短打安打 | BH | Bunt Hit |

#### 出局類型
| 類型 | 代碼 | 說明 |
|------|------|------|
| 滾地球出局 | GO | Ground Out |
| 飛球出局 | FO | Fly Out |
| 三振 | K | Strikeout（被看三振用 Ⓚ） |
| 接殺（飛球） | F | Fly Out（含位置） |
| 接殺（界外飛球） | FF | Foul Fly Out |
| 內野飛球 | IFFY | Infield Fly |
| 封殺 | FC | Fielder's Choice |
| 雙殺 | DP | Double Play |
| 三殺 | TP | Triple Play |
| 觸殺 | TAG | Tag Out |
| 犧牲短打 | SAC | Sacrifice Bunt |
| 犧牲飛球 | SF | Sacrifice Fly |
| 打擊妨礙 | INT | Interference |

#### 其他打擊結果
| 類型 | 代碼 | 說明 |
|------|------|------|
| 野手選擇 | FC | Fielder's Choice |
| 失誤上壘 | E | Error（需記錄失誤守備員） |
| 四壞保送 | BB | Base on Balls |
| 雙殺打 | GDP | Ground into Double Play |

### 5.4 打擊方向記錄

使用棒球場示意圖，讓記錄員**點擊或滑動**選擇打擊落點方向：

```
守備位置代號：
1 = 投手 (P)
2 = 捕手 (C)
3 = 一壘手 (1B)
4 = 二壘手 (2B)
5 = 三壘手 (3B)
6 = 游擊手 (SS)
7 = 左外野手 (LF)
8 = 中外野手 (CF)
9 = 右外野手 (RF)
```

**方向區域劃分：**
- 內野：投手前方、一壘線、二壘方向、游擊方向、三壘線
- 外野：左外野線、左中外野、中外野、右中外野、右外野線
- 界外區：左側界外、右側界外、本壘後方

**打擊球質分類：**
| 符號 | 說明 |
|------|------|
| G | 滾地球 (Ground Ball) |
| L | 平飛球 (Line Drive) |
| F | 飛球 (Fly Ball) |
| P | 小飛球 (Pop Up) |
| BU | 短打 (Bunt) |

### 5.5 守備路徑記錄
記錄守備傳球路徑，例如：
- `6-3`：游擊接球傳一壘刺殺
- `4-6-3`：二壘手接球、傳游擊、轉傳一壘（雙殺）
- `5-4-3`：三壘手接球、傳二壘、轉傳一壘（雙殺）
- `F8`：中外野手接殺飛球
- `E6`：游擊手失誤

| 跑壘失誤 | ROE | Running Error |

- 滑動跑者到目標壘包

## 9. 比分與局面管理

### 9.1 計分看板
```
       1  2  3  4  5  6  7  8  9  R  H  E
客隊   0  1  0  0  2  0  0  1  0  4  8  1
主隊   1  0  0  3  0  0  0  0  X  4  7  0
```

### 9.2 局面管理
- 自動偵測三出局換邊
- 半局結束自動切換攻守
- 延長賽自動新增局數
- 提前結束比賽（如提前結束規則）

## 10. 修改與回溯功能

### 10.1 撤銷/重做
- 支援**逐步撤銷**（Undo），最多回溯 50 步
- 支援**重做**（Redo）
- 操作歷史記錄清單

### 10.2 直接修改
- 點擊歷史記錄中的任一打席，可直接修改
- 修改後自動重新計算後續狀態
- 修改標記（顯示哪些記錄被修改過）

### 10.3 自動儲存
- 每次操作後自動儲存至 LocalStorage
- 支援多場比賽紀錄共存
- 瀏覽器關閉後不會遺失資料

## 13. 非功能性需求

### 13.1 使用者體驗
- 大型按鈕，最小觸控區域 44px × 44px
- 操作回饋（觸覺振動 + 視覺回饋）
- 常用操作不超過 2 次點擊完成
- 支援左手/右手操作模式
- 夜間模式（夜間比賽照明不足）

### 13.2 可靠性
- 自動儲存，無需手動存檔
- 離線完全可用
- 電量餘額提醒（低電量時減少動畫）

### 13.3 安全性
- 資料存儲在本機，不傳輸至外部
- 匯出檔案不含敏感個資（可選遮蔽姓名）
