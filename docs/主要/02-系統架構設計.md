## 3. 核心模組設計

### 3.1 GameEngine（比賽引擎）— 狀態機

比賽引擎是整個系統的核心，負責管理比賽狀態的流轉：

```
                    ┌─────────┐
                    │  SETUP  │  比賽設定中
                    └────┬────┘
                         │ startGame()
                         ▼
                    ┌─────────┐
              ┌────▶│ AT_BAT  │◀───── 下一位打者
              │     └────┬────┘
              │          │ recordPitch()
              │          ▼
              │     ┌─────────┐
              │     │ PITCHING│  等待投球結果
              │     └────┬────┘
              │          │
              │     ┌────┴────┐
              │     ▼         ▼
              │ ┌───────┐ ┌────────┐
              │ │ COUNT │ │IN_PLAY │  球打出去
              │ │UPDATE │ └────┬───┘
              │ └───┬───┘      │ recordResult()
              │     │          ▼
              │     │     ┌─────────┐
              │     │     │ RESULT  │  記錄打擊結果
              │     │     └────┬────┘
              │     │          │
              │     │     ┌────┴────┐
              │     │     ▼         ▼
              │     │ ┌───────┐ ┌────────┐
              │     │ │ OUTS  │ │ON_BASE │  上壘
              │     │ │ 出局  │ └────┬───┘
              │     │ └───┬───┘      │
              │     │     │          │
              │     ▼     ▼          │
              │  ┌──────────────┐    │
              │  │ CHECK_STATE  │◀───┘
              │  └──────┬───────┘
              │         │
              │    ┌────┴────┐
              │    ▼         ▼
              │ ┌──────┐ ┌─────────┐
              └─┤NEXT  │ │HALF_END │  半局結束
                │BATTER│ └────┬────┘
                └──────┘      │
                         ┌────┴────┐
                         ▼         ▼
                    ┌─────────┐ ┌─────────┐
                    │NEXT_HALF│ │GAME_END │  比賽結束
                    └────┬────┘ └─────────┘
                         │
                         └──▶ 回到 AT_BAT
```

### 3.2 PlayRecorder（逐球記錄器）

負責記錄每一球的詳細資訊：

```javascript
// 記錄資料結構概念
PlayRecord = {
    id:           "唯一識別碼",
    gameId:       "所屬比賽",
    inning:       "局數",
    halfInning:   "上/下半局",
    atBatNumber:  "第幾個打席",
    pitchNumber:  "該打席第幾球",
    pitcher:      "投手資訊",
    batter:       "打者資訊",
    pitchResult:  "該球結果（S/B/F/IP...）",
    hitResult:    "打擊結果（1B/GO/HR...）",
    hitDirection: "打擊方向（區域代碼）",
    hitType:      "球質（G/L/F/P）",
    fieldingPath: "守備路徑（6-3）",
    runners:      "跑壘變化",
    timestamp:    "記錄時間"
}
```

### 3.3 UndoManager（撤銷管理器）

```
操作堆疊：
┌─────────────────────┐
│ Undo Stack (最多50)  │
│ ┌─────────────────┐ │
│ │ Action N        │ │ ◀── 最新操作
│ │ Action N-1      │ │
│ │ Action N-2      │ │
│ │ ...             │ │
│ │ Action 1        │ │ ◀── 最早操作
│ └─────────────────┘ │
└─────────────────────┘

┌─────────────────────┐
│ Redo Stack          │
│ ┌─────────────────┐ │
│ │ （撤銷後的操作）  │ │
│ └─────────────────┘ │
└─────────────────────┘
```

每個 Action 包含：
- `type`：操作類型
- `before`：操作前狀態快照
- `after`：操作後狀態快照
- `timestamp`：操作時間

### 3.4 RunnerManager（跑壘管理器）

管理壘包上的跑者狀態：

```
              二壘
              [2B]
             /    \
            /      \
        三壘        一壘
        [3B]       [1B]
            \      /
             \    /
              本壘
              [HP]
```

- 追蹤每個壘包上的跑者 ID
- 處理推進邏輯（安打、暴投、盜壘等）
- 計算得分（跑者回到本壘）
- 處理強迫進壘與非強迫進壘

## 4. 路由設計（SPA）

使用 Hash Router 實現單頁應用：

| 路由 | 頁面 | 說明 |
|------|------|------|
| `#/` | 首頁 | 比賽列表 / 新增比賽 |
| `#/setup/:id` | 比賽設定 | 設定隊伍、球員、打序 |
| `#/live/:id` | 即時記錄 | 主要記錄操作介面 |
| `#/stats/:id` | 統計報表 | 比賽統計資料 |
| `#/history/:id` | 歷史記錄 | 逐球回顧 |
| `#/settings` | 系統設定 | 偏好設定 |

## 5. 資料儲存策略

### 5.1 儲存分層

| 資料類型 | 儲存方式 | 說明 |
|----------|----------|------|
| 比賽設定 | LocalStorage | 快速存取 |
| 逐球記錄（進行中） | LocalStorage | 即時自動儲存 |
| 完成的比賽 | IndexedDB | 資料量較大 |
| 使用者偏好 | LocalStorage | 主題、操作偏好 |
| 匯出檔案 | File API | 下載為 JSON/CSV |

### 5.2 自動儲存機制
```
每次操作 → 更新記憶體中的 Game State
         → 同步寫入 LocalStorage（去抖動 300ms）
         → 每 30 秒完整備份至 IndexedDB
```

### 5.3 資料大小估算
- 一場 9 局比賽約 300~500 個 Play 記錄
- 每個 Play 約 200~500 bytes
- 一場比賽完整資料約 100~250 KB
- LocalStorage 上限 5~10 MB（約可存 20~50 場）
- IndexedDB 幾乎無上限

## 6. 手勢與互動設計

### 6.1 觸控手勢

| 手勢 | 功能 | 使用場景 |
|------|------|----------|
| 單點觸控 | 選擇 / 確認 | 通用操作 |
| 長按 | 修改 / 查看詳情 | 修改歷史記錄 |
| 左滑 | 下一步 / 切換面板 | 面板導航 |
| 右滑 | 上一步 / 返回 | 面板返回 |
| 上滑 | 展開詳細選項 | 打擊結果展開 |
| 下滑 | 收合面板 | 關閉浮層 |
| 捏合縮放 | 球場圖縮放 | 球場方向圖 |
| 拖曳 | 跑者移動 / 排序 | 壘包、打序 |

### 6.2 回饋機制
- **視覺**：按鈕按壓效果、顏色變化、動畫過渡
- **振動**：操作確認短振動（50ms）、重要事件長振動（200ms）
- **音效**：可選的操作音效（如三振、全壘打）

## 7. PWA 支援

### 7.1 Service Worker 策略
- **Cache First**：靜態資源（HTML、CSS、JS、圖片）
- **Network First**：無（純前端無後端 API）
- 離線時完全可用

## 8. 效能優化策略

| 策略 | 做法 |
|------|------|
| CSS 分割 | 依模組拆分 CSS，減少單檔過大 |
| JS 延遲載入 | 統計模組等非關鍵模組延遲載入 |
| DOM 操作優化 | 使用 DocumentFragment 批次更新 |
| 事件委派 | 父元素統一監聽事件 |
| 記憶體管理 | 及時清除不用的事件監聽器 |
| 動畫性能 | 使用 transform/opacity 做 GPU 加速 |
| 圖片 | SVG 球場圖，體積小且支援縮放 |
